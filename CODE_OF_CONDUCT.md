# Code of Conduct

## Our principles

This project welcomes contributors and users from all backgrounds. Participation is governed by three simple principles:

1. **Be respectful.** Disagree with ideas, not with people. Assume good faith. Criticise the work, not the author.
2. **Stay on-topic.** Discussions should relate to the standard, the website, the tooling, or the practice of solution architecture. Off-topic political, religious, or personal content is out of scope.
3. **Be useful.** Contributions should improve the standard or the community around it. Venting, trolling, and low-effort engagement are not contributions.

## Scope

This Code of Conduct applies to all project spaces: the GitHub repository, Issues, Pull Requests, Discussions, and any official communication channel.

## Unacceptable behaviour

- Harassment, intimidation, or personal attacks
- Discriminatory language or imagery
- Sustained disruption of discussions
- Publishing private information without consent
- Spam, advertising, or promotional content

## Reporting

If you experience or witness behaviour contrary to this Code of Conduct, please contact the maintainer privately via the contact form on [andi.me](https://andi.me/).

Reports are handled confidentially. The maintainer will respond within seven days and take proportionate action, which may include:

- A private warning
- A public correction
- A temporary or permanent ban from project spaces

## Attribution

This Code of Conduct is intentionally brief and principles-based. It is adapted from the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/) and the [Recurse Center Social Rules](https://www.recurse.com/manual#sub-sec-social-rules), but simpler.

For a comprehensive code of conduct, many projects adopt the full [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) — this project does not require that level of formality but welcomes its principles.
