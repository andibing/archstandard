Download a blank SAD template conforming to ADS v1.0.0 in your preferred format. All templates contain the full section structure with placeholder fields ready to complete.

```bash
# Using Pandoc (recommended)
pandoc sad-template.md -o sad-template.docx --reference-doc=template.docx

# Using a custom script with the JSON/YAML template
# (See the schema documentation for programmatic generation)
```

A dedicated Word template generator is planned for a future release of the standard.
</Aside>

## Validation

The JSON and YAML templates can be validated against the [ADS JSON Schema](/standard/schema/):

```bash
# Validate JSON template
npx ajv validate -s schema/ads.schema.json -d my-sad.json

# Validate YAML (convert to JSON first)
npx js-yaml my-sad.yaml | npx ajv validate -s schema/ads.schema.json -d /dev/stdin
```

## Organisation Customisation

To create an organisation-specific template:

1. Download the YAML or JSON template
2. Add your `organisationProfile` section with internal tool mappings and governance gates
3. Add any `customSections` required by your organisation
4. Distribute the customised template to your architecture teams

See [Conformance and Usage](/standard/how-to-use/) for customisation guidelines.
