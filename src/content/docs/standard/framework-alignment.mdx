## Alignment Model

ADS maps to multiple established frameworks. This page documents the traceability between each section of the standard and its source frameworks.

## ISO/IEC 42010 Alignment

The standard follows the ISO 42010 conceptual model:

| ISO 42010 Concept | ADS Mapping |
|-------------------|---------------------|
| System of Interest | The solution being documented |
| Stakeholders | Stakeholders & Concerns |
| Concerns | Concerns Matrix |
| Architecture Viewpoint | Defined viewpoints |
| Architecture View | The content within each view |
| Model Kind | Diagrams, tables, and descriptions within views |
| Correspondences | Cross-references between views and quality attribute refs |
| Architecture Description | The complete Solution Architecture Document |

## 4+1 View Model Alignment

| 4+1 Original View | ADS View | Key Content |
|-------------------|---------------------|-------------|
| Logical View | Logical View | Component architecture, service mapping |
| Process View | Integration & Data Flow View *(adapted)* | Data flows, integrations, interfaces |
| Development View | Lifecycle Management | CI/CD, SDLC, tooling |
| Physical View | Physical View | Deployment, infrastructure, networking |
| +1 Scenarios | Scenarios | Use cases, ADRs |
| *(Extended)* | Data View | Data architecture (from RM-ODP Information viewpoint) |
| *(Extended)* | Security View | Security architecture (cross-cutting in 4+1, elevated to full view) |

:::note
The standard extends the classic 4+1 model with dedicated **Data** and **Security** views. In the original 4+1 model, these are cross-cutting concerns spread across all views. Modern solution architecture demands they be treated as first-class views due to regulatory, compliance, and data governance requirements. The Process View has been adapted to focus on integration and data flows rather than concurrency.
:::
