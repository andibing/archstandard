## Alignment Model

The ADS-001 uses **ISO/IEC 42010** as its meta-model and maps to multiple established frameworks. This page documents the traceability between each section of the standard and its source frameworks.

## ISO/IEC 42010 Alignment

The standard follows the ISO 42010 conceptual model:

| ISO 42010 Concept | ADS-001 Mapping |
|-------------------|---------------------|
| System of Interest | The solution being documented |
| Stakeholders | Section 2: Stakeholders & Concerns |
| Concerns | Section 2: Concerns Matrix |
| Architecture Viewpoint | Section 3: Six defined viewpoints |
| Architecture View | The content within each Section 3.x |
| Model Kind | Diagrams, tables, and descriptions within views |
| Correspondences | Cross-references between views and Quality Pillar refs |
| Architecture Description | The complete Solution Architecture Document |

## 4+1 View Model Alignment

| 4+1 View | ADS-001 Section | Key Content |
|----------|---------------------|-------------|
| Logical View | 3.1 Logical View | Component architecture, service mapping |
| Process View | 3.2 Process View | Data flows, integrations, interfaces |
| Development View | 5. Lifecycle Management | CI/CD, SDLC, tooling |
| Physical View | 3.3 Physical View | Deployment, infrastructure, networking |
| +1 Scenarios | 3.6 Scenarios | Use cases, ADRs |
| *(Extended)* | 3.4 Data View | Data architecture (from RM-ODP Information viewpoint) |
| *(Extended)* | 3.5 Security View | Security architecture (cross-cutting in 4+1, elevated to full view) |

:::note
The standard extends the classic 4+1 model with dedicated **Data** and **Security** views. In the original 4+1 model, these are cross-cutting concerns spread across all views. Modern enterprise architecture demands they be treated as first-class views due to regulatory, compliance, and data governance requirements.
:::
