## Purpose

The Integration & Data Flow View describes how data moves through the solution, how components communicate, and how the solution integrates with internal and external systems. It addresses the concerns of integrators, operations teams, and architects.

<Aside type="note" title="Naming">
This view is adapted from Kruchten's **Process View**, which originally focused on concurrency and runtime process decomposition. In modern solution architecture, the primary concerns at this level are data flows, integration patterns, and interface contracts. The name has been updated to reflect this contemporary usage.
</Aside>
