## Scope

This standard defines the required and recommended structure for documenting the architecture of technology solutions. It establishes a common framework for High Level Design (HLD) and Solution Architecture Description (SAD) documents that is:

- **Organisation-agnostic** — adoptable by any enterprise without modification
- **Platform-neutral** — applicable to cloud, on-premises, hybrid, and SaaS architectures
- **Framework-aligned** — traceable to ISO/IEC 42010, the 4+1 Architectural View Model, and the AWS and Azure Well-Architected Frameworks
- **Schema-driven** — backed by a formal JSON Schema enabling validation and multi-format generation

## Normative References

This standard draws upon and aligns with the following frameworks and specifications:

| Reference | Title | Usage |
|-----------|-------|-------|
| ISO/IEC/IEEE 42010:2022 | Architecture Description | Meta-model for stakeholders, concerns, viewpoints, and views |
| Kruchten 4+1 | Architectural View Model | Six-view structure (extended with Data and Security views) |
| AWS Well-Architected | Framework (6 Pillars) | Quality pillar definitions and evaluation criteria |
| Azure Well-Architected | Framework (5 Pillars) | Quality pillar definitions with tradeoff documentation |
| TOGAF | The Open Group Architecture Framework | Architecture domain alignment |

## Terms and Definitions

The key words "SHALL", "SHOULD", "MAY", and "OPTIONAL" in this standard are to be interpreted as follows:

- **SHALL** — An absolute requirement. The section must be completed.
- **SHOULD** — Recommended. The section should be completed unless there is a justified reason not to.
- **MAY** / **OPTIONAL** — Truly optional. Include if relevant to the solution.

These terms are mapped to the standard's maturity levels:

| Maturity Level | Keyword | Meaning |
|---------------|---------|---------|
| **Minimum** | SHALL | Required for all designs |
| **Recommended** | SHOULD | Required for production-grade designs |
| **Comprehensive** | MAY | For mature, enterprise-grade documentation |
