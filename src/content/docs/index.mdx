## Core Principles

<CardGrid stagger>
	<Card title="View-Model Aligned" icon="open-book">
		Organized around architectural viewpoints (Logical, Process, Physical, Data, Security, Scenarios) aligned with Kruchten's 4+1 model and ISO/IEC 42010.
	</Card>
	<Card title="Quality Pillars as Cross-Cuts" icon="approve-check-circle">
		Operational Excellence, Reliability, Performance, Cost, and Sustainability are evaluation lenses applied across all views - inspired by AWS and Azure Well-Architected Frameworks.
	</Card>
	<Card title="Schema-Driven" icon="setting">
		A formal JSON Schema enables validation and generation of HLD documents in multiple formats: Markdown, Word, or web pages.
	</Card>
	<Card title="Maturity Levels" icon="up-caret">
		Every section is tagged as Minimum, Recommended, or Comprehensive - enabling progressive adoption from lightweight designs to full enterprise documentation.
	</Card>

## Framework Alignment

This standard synthesizes and aligns with established architecture frameworks:

| Framework | How It Is Used |
|-----------|---------------|
| **ISO/IEC 42010** | Meta-model: stakeholders, concerns, viewpoints, views, correspondences |
| **4+1 Architectural View Model** | Six architectural views structure (Logical, Process, Physical, Data, Security + Scenarios) |
| **AWS Well-Architected Framework** | Six quality pillars as cross-cutting evaluation lenses |
| **Azure Well-Architected Framework** | Five pillars with explicit tradeoff documentation |
| **TOGAF** | Architecture domains (Business, Data, Application, Technology) inform view content |
