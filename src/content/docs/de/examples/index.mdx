## Lernen am Beispiel

Diese Beispiel-SADs zeigen, wie der Architecture Description Standard auf reale Lösungen angewendet wird. Jedes Beispiel ist ein vollständig (fiktiv) ausgefülltes Solution Architecture Document, das zeigt, wie ein gutes SAD über verschiedene Projekttypen hinweg aussieht.

<Aside type="tip" title="Keine echten Projekte">
Diese Beispiele sind fiktiv, aber realistisch. Sie sollen die Abschnitte, Felder und Bewertungen des Standards veranschaulichen — und nicht bestimmte Technologieentscheidungen vorschreiben.
</Aside>

## Wie Sie diese Beispiele nutzen

1. **Lesen Sie das Beispiel, das zu Ihrem Projekttyp passt** — wählen Sie das Beispiel, das Ihrer Lösung am nächsten kommt
2. **Beachten Sie die Tiefe** — jedes Beispiel nennt seine Dokumentationstiefe (Minimum, Recommended oder Comprehensive)
3. **Prüfen Sie die Bewertung** — jedes Beispiel enthält eine Bewertungstabelle für die Konformität
4. **Übernehmen Sie die Struktur** — laden Sie eine [leere Vorlage](/de/standard/templates/) herunter und verwenden Sie das Beispiel als Referenz beim Ausfüllen Ihres eigenen SAD

## Ihr SAD gegen das Schema validieren

Sobald Sie Ihr SAD im JSON- oder YAML-Format fertiggestellt haben, validieren Sie es gegen das ADS-Schema:

```bash
# Ein JSON-SAD gegen das gehostete Schema validieren
npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d my-sad.json

# Ein YAML-SAD validieren (zuerst in JSON konvertieren)
npx js-yaml my-sad.yaml | npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d /dev/stdin

# Mit Python
pip install jsonschema pyyaml
python -m jsonschema -i my-sad.json https://archstandard.org/schema/v1.0.0/ads.schema.json
```

Das Schema validiert Struktur, Pflichtfelder und Enum-Werte — und stellt sicher, dass Ihr SAD dem Standard entspricht. Vollständige Details finden Sie auf der [JSON-Schema-Seite](/de/standard/schema/).
