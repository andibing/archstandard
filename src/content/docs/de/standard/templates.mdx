## Eine Vorlage beziehen

Laden Sie eine leere SAD-Vorlage von der **[Downloads](/de/standard/downloads/)**-Seite in Ihrem bevorzugten Format (Word, Markdown, YAML oder JSON) herunter.

## Drei Wege, Vorlagen zu verwenden

**1. Manuell ausfüllen** — öffnen Sie die Vorlage in Ihrem Editor, Wiki oder Textverarbeitungsprogramm und füllen Sie jeden Abschnitt aus.

**2. In ein LLM importieren** — übergeben Sie die Vorlage an ChatGPT, Claude oder Copilot zusammen mit einem Projekt-Briefing, um einen strukturierten Erstentwurf zu erzeugen.

**3. In Confluence einfügen** — kopieren Sie die Markdown-Vorlage in Ihre Zwischenablage und fügen Sie sie direkt in Confluence, Notion oder ein beliebiges Wiki ein, das Markdown unterstützt.

## Zwischen Formaten konvertieren

Verwenden Sie [Pandoc](https://pandoc.org/), um zwischen Formaten zu konvertieren:

```bash
# Markdown nach Word
pandoc sad-template.md -o my-sad.docx

# Markdown nach ODF (LibreOffice)
pandoc sad-template.md -o my-sad.odt

# Mit individueller Markendarstellung
pandoc sad-template.md -o my-sad.docx --reference-doc=your-template.docx
```

## Ihr SAD validieren

Validieren Sie ein fertiggestelltes SAD (JSON oder YAML) gegen das ADS-Schema:

```bash
npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d my-sad.json
```

Weitere Validierungsoptionen finden Sie auf der Seite [JSON Schema](/de/standard/schema/).

## Anpassung für Ihre Organisation

Sie können den Standard für Ihre Organisation erweitern, ohne die Kernabschnitte zu verändern. Es gibt zwei Erweiterungspunkte:

- **`organisationProfile`** — bildet den Standard auf Ihre internen Werkzeuge, Standards und Governance-Gates ab
- **`customSections`** — fügt organisationsspezifische Abschnitte hinzu (z. B. Cloud-Governance-Checklisten, Datenschutz-Folgenabschätzungen)

### Wie Sie diese zu Ihrem SAD hinzufügen

**In YAML oder JSON:** Fügen Sie die Blöcke `organisationProfile` und/oder `customSections` am Ende Ihrer SAD-Datei (nach `appendices`) hinzu. Sie stehen auf der obersten Ebene, neben den Standardabschnitten.

**In Markdown:** Fügen Sie neue Überschriften nach Abschnitt 7 (Anhänge) mit Ihrem benutzerdefinierten Inhalt hinzu. Verwenden Sie dasselbe Tabellenformat wie im Rest des Dokuments.

**In Word:** Fügen Sie neue Abschnitte nach den Anhängen hinzu. Folgen Sie demselben Überschriften- und Tabellenstil.

**Für Ihre gesamte Organisation:** Erstellen Sie eine angepasste Vorlage mit vorausgefülltem `organisationProfile` und verteilen Sie sie an Ihre Architektur-Teams. So beginnt jedes neue SAD mit den richtigen Werkzeug-Mappings und Governance-Gates.

### Beispiel Organisationsprofil (YAML)

Fügen Sie diesen Block am Ende Ihres YAML-SAD hinzu, um den Standard auf die Werkzeuge und Governance Ihrer Organisation abzubilden:

```yaml
organisationProfile:
  organisationName: "Acme Corp"
  tooling:
    cmdb: "ServiceNow"
    secretStore: "HashiCorp Vault"
    monitoring: "Datadog"
    siem: "Splunk"
    cicd: "GitHub Actions"
    portfolioManagement: "LeanIX"
  internalStandards:
    - id: "STD-001"
      name: "Cloud Platform Standards"
      version: "2.0"
      mappedSections: ["physicalView", "securityView"]
    - id: "POL-008"
      name: "Information Security Policy"
      version: "4.1"
      mappedSections: ["securityView", "dataView"]
  governanceGates:
    - gateName: "Dev/Test Review"
      requiredMaturityLevel: "minimum"
      requiredSections: ["documentControl", "executiveSummary", "architecturalViews", "riskGovernance"]
    - gateName: "Production Approval"
      requiredMaturityLevel: "recommended"
      requiredSections: ["documentControl", "executiveSummary", "stakeholders", "architecturalViews", "qualityAttributes", "lifecycleManagement", "riskGovernance"]
    - gateName: "Enterprise Architecture Review"
      requiredMaturityLevel: "comprehensive"
      requiredSections: ["documentControl", "executiveSummary", "stakeholders", "architecturalViews", "qualityAttributes", "lifecycleManagement", "riskGovernance", "appendices"]
```

### Beispiel benutzerdefinierte Abschnitte

Fügen Sie organisationsspezifische Abschnitte hinzu, ohne den Kernstandard zu verändern:

```yaml
customSections:
  - id: "cloud-governance"
    title: "Cloud-Governance-Checkliste"
    description: "Organisationsspezifische Cloud-Governance-Anforderungen"
    maturityLevel: "recommended"
    parentSection: "physicalView"
    content: |
      | Kontrolle | Status | Nachweis |
      |-----------|--------|----------|
      | Tagging-Richtlinie angewendet | Ja / Nein | |
      | Kostenwarnungen konfiguriert | Ja / Nein | |
      | Backup-Richtlinie angewendet | Ja / Nein | |

  - id: "data-privacy"
    title: "Datenschutz-Folgenabschätzung"
    description: "DSGVO- und datenschutzspezifische Anforderungen"
    maturityLevel: "recommended"
    parentSection: "dataView"
    content: |
      | Frage | Antwort |
      |-------|---------|
      | DSFA durchgeführt? | Ja / Nein |
      | Auftragsverarbeitungsverträge vorhanden? | Ja / Nein |
      | Auskunftsanfrage-Prozess dokumentiert? | Ja / Nein |
```

Vollständige Anpassungsrichtlinien finden Sie unter [Konformität und Verwendung](/de/standard/how-to-use/).
