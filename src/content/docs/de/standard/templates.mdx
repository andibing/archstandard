## Herunterladbare Vorlagen

Laden Sie eine leere SAD-Vorlage gemäß ADS v1.1.0 in Ihrem bevorzugten Format herunter. Alle Vorlagen enthalten die vollständige Abschnittsstruktur mit Platzhalterfeldern, die ausgefüllt werden können.

### Maschinenlesbare Formate

#### Markdown-Vorlage

Vollständige Vorlage im Markdown-Format. Ideal für Versionskontrolle neben Code in Git-Repositories.

**[Download sad-template.md](/templates/sad-template.md)**

#### YAML-Vorlage

Strukturierte Vorlage im YAML-Format. Ideal für programmatische Verarbeitung, CI/CD-Integration und Tooling.

**[Download sad-template.yaml](/templates/sad-template.yaml)**

#### JSON-Vorlage

Strukturierte Vorlage im JSON-Format. Validiert gegen das ADS JSON Schema.

**[Download sad-template.json](/templates/sad-template.json)**

### Vollständiger Standard zum Download

#### Vollständiger Standard (DOCX)

Der vollständige ADS-Standard als einzelnes Word-Dokument mit Inhaltsverzeichnis. Geeignet für Offline-Lektüre, Governance-Einreichungen und Druck.

**[Download ads-standard-v1.docx](/ads-standard-v1.docx)**

### Menschenlesbare Formate

#### Word / DOCX

Generieren Sie ein Word-Dokument aus der Markdown-Vorlage mit [Pandoc](https://pandoc.org/). Ideal für formale Governance-Einreichungen und Offline-Prüfung.

Siehe den Pandoc-Befehl unten zur Generierung.

#### Confluence / Wiki

Kopieren Sie die Markdown-Vorlage in Ihre Zwischenablage und fügen Sie sie direkt in Confluence, Notion oder jedes Wiki ein, das Markdown unterstützt.

Um die Markdown-Vorlage zu kopieren, laden Sie die Datei sad-template.md herunter und kopieren Sie den Inhalt.

> **Tipp: Word / DOCX generieren**
>
> Konvertieren Sie die Markdown-Vorlage mit [Pandoc](https://pandoc.org/) in das Word-Format:
>
> ```bash
> # Basic conversion
> pandoc sad-template.md -o sad-template.docx
>
> # With a custom reference document for branding/styling
> pandoc sad-template.md -o sad-template.docx --reference-doc=your-template.docx
> ```
>
> Ein dedizierter Word-Vorlagengenerator ist für eine zukünftige Version des Standards geplant.

## Validierung

Die JSON- und YAML-Vorlagen können gegen das [ADS JSON Schema](/de/standard/schema/) validiert werden:

```bash
# Validate JSON template
npx ajv validate -s schema/ads.schema.json -d my-sad.json

# Validate YAML (convert to JSON first)
npx js-yaml my-sad.yaml | npx ajv validate -s schema/ads.schema.json -d /dev/stdin
```

## Organisationsanpassung

Um eine organisationsspezifische Vorlage zu erstellen:

1. Laden Sie die YAML- oder JSON-Vorlage herunter
2. Fügen Sie Ihren `organisationProfile`-Abschnitt mit internen Tool-Zuordnungen und Governance Gates hinzu
3. Fügen Sie alle `customSections` hinzu, die Ihre Organisation benötigt
4. Verteilen Sie die angepasste Vorlage an Ihre Architekturteams

Siehe [Konformität und Nutzung](/de/standard/how-to-use/) für Anpassungsrichtlinien.
