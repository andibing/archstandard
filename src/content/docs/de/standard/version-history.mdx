## Versionierungsrichtlinie

Der Architecture Description Standard (ADS) verwendet [**semantische Versionierung**](https://semver.org/):

- **MAJOR**-Version - Strukturelle Änderungen, die die Abwärtskompatibilität brechen (z.B. Abschnitte reorganisiert, entfernt oder grundlegend geändert)
- **MINOR**-Version - Neue Abschnitte, Ergänzungen von Leitlinien oder nicht-brechende Verbesserungen
- **PATCH**-Version - Korrekturen, Klarstellungen, Tippfehler-Behebungen

## Frühere Versionen

Frühere Hauptversionen sind dauerhaft unter versionierten URLs verfügbar. SADs, die auf eine frühere Version verweisen, bleiben gültig.

## Releases

### v1.0.0 - Erstveröffentlichung

**Datum:** 2026-03-27

Die erste Version des Architecture Description Standard (ADS), die jahrelange praktische Erfahrung in der Lösungsarchitektur mit etablierten Frameworks zusammenführt:

- **ISO/IEC 42010** - Architecture Description Meta-Modell (Stakeholder, Concerns, Viewpoints, Views)
- **4+1 Architectural View Model** - Sechs Architekturansichten (erweitert von 5 mit dedizierten Data und Security Views)
- **Cloud Well-Architected Frameworks** - Qualitätsattribute abgeleitet von AWS, Azure, GCP, Oracle und IBM Well-Architected Frameworks
- **TOGAF** - Ausrichtung an Architekturdomänen (Business, Data, Application, Technology)
- **Branchenübliche SAD/HLD-Vorlagen** - Best Practices von arc42, ServiceNow, NHS SAF, ALMBoK und Unternehmensvorlagen

**Struktur:**
- 8 Top-Level-Abschnitte (0-7)
- 6 Architekturansichten
- 5 Qualitätsattribute
- 3 Documentation Depths (Minimum, Recommended, Comprehensive)
- JSON Schema für Multi-Format-Generierung
- Framework-Alignment-Rückverfolgbarkeit

### v1.1 - Beispiele und Leitlinien

**Datum:** 2026-03-31

- Vier vollständige Beispiel-SADs (Employee Directory, Customer API Platform, Cloud Migration, archstandard.org)
- Bewertungshinweise (1/3/5) zu allen Abschnittsseiten hinzugefügt
- Verbesserungen für Barrierefreiheit und Neurodiversität
- Französische und deutsche Übersetzungen
- Vorlagen-Generator-Skript (Schema als Single Source of Truth)
