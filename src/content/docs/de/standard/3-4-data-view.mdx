## Zweck

Die Datenansicht beschreibt, wie Daten gespeichert, klassifiziert, geschutzt und über ihren gesamten Lebenszyklus verwaltet werden. Sie adressiert die Anliegen von Datenarchitekten, DBAs, Compliance-Beauftragten und Datenschutzteams.

| *[Name]* | *[z.B. PostgreSQL, S3, Redis]* | Ja / Nein | *[Zeitraum]* | *[Grosse]* | Public / Internal / Restricted / Highly Restricted | Ja / Nein | None / Storage / Container / Application | *[Methode]* |

<div class="guidance-box">
<h4>Leitfaden</h4>

Dokumentieren Sie für jeden Datenspeicher:
- **Speichertechnologie** — Die spezifische Technologie (z.B. SQL Server, S3 Bucket, Azure Blob)
- **Autoritativer Datenspeicher** — Ist dies die Haupt-/autoritative Quelle der Daten?
- **Aufbewahrungsfrist** — Wie lange werden Daten vor der Loschung aufbewahrt?
- **Sicherheitsklassifizierung** — Hochste Klassifizierung der Daten im Speicher
- **Personenbezogene Daten** — Enthalt er PII oder besonders schutzwurdige personenbezogene Daten (SPI)?
- **Verschlusselung** — Angewandte Verschlusselungsebene (Storage, Container, Application)
- **Schlusselverwaltung** — Wie Verschlusselungsschlussel verwaltet werden (anbieterverwaltert, HSM, kundenspezifisch)
</div>
