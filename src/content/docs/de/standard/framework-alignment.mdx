## Ausrichtungsmodell

ADS verwendet **ISO/IEC 42010** als Meta-Modell und ordnet sich mehreren etablierten Frameworks zu. Diese Seite dokumentiert die Rückverfolgbarkeit zwischen jedem Abschnitt des Standards und seinen Quell-Frameworks.

## ISO/IEC 42010-Ausrichtung

Der Standard folgt dem konzeptionellen Modell von ISO 42010:

| ISO 42010-Konzept | ADS-Zuordnung |
|-------------------|---------------|
| System of Interest | Die dokumentierte Lösung |
| Stakeholders | Abschnitt 2: Stakeholder & Anliegen |
| Concerns | Abschnitt 2: Anliegen-Matrix |
| Architecture Viewpoint | Abschnitt 3: Sechs definierte Blickwinkel |
| Architecture View | Der Inhalt innerhalb jedes Abschnitts 3.x |
| Model Kind | Diagramme, Tabellen und Beschreibungen innerhalb der Ansichten |
| Correspondences | Querverweise zwischen Ansichten und Qualitätsattribut-Referenzen |
| Architecture Description | Das vollständige Solution Architecture Document |

## 4+1 View Model-Ausrichtung

| 4+1 Originalansicht | ADS-Abschnitt | Wesentlicher Inhalt |
|---------------------|---------------|---------------------|
| Logical View | 3.1 Logische Ansicht | Komponentenarchitektur, Dienstzuordnung |
| Process View | 3.2 Integrations- & Datenflussansicht *(angepasst)* | Datenflusse, Integrationen, Schnittstellen |
| Development View | 5. Lebenszyklusmanagement | CI/CD, SDLC, Werkzeuge |
| Physical View | 3.3 Physische Ansicht (Infrastruktur) | Bereitstellung, Infrastruktur, Netzwerk |
| +1 Scenarios | 3.6 Szenarien | Anwendungsfalle, ADRs |
| *(Erweitert)* | 3.4 Datenansicht | Datenarchitektur (aus dem RM-ODP Information Viewpoint) |
| *(Erweitert)* | 3.5 Sicherheitsansicht | Sicherheitsarchitektur (querschnittlich in 4+1, zur vollständigen Ansicht erhoben) |

:::note
Der Standard erweitert das klassische 4+1-Modell um eigene **Daten-** und **Sicherheitsansichten**. Im ursprunglichen 4+1-Modell sind dies querschnittliche Anliegen, die über alle Ansichten verteilt sind. Moderne Lösungsarchitektur erfördert, dass sie aufgrund regulatorischer, Compliance- und Daten-Governance-Anforderungen als erstklassige Ansichten behandelt werden. Die Process View würde angepasst, um sich auf Integration und Datenflusse statt auf Nebenlaufigkeit zu konzentrieren.
:::
