## 5.2 Transition de service et migration

<span class="maturity-indicator maturity-recommended">Recommandé</span>

### Classification de la migration (6 R)

Si cette solution remplace ou migre un système existant, classifiez l'approche de migration :

| Classification | Sélectionné ? | Description |
|---------------|-----------|-------------|
| **Retain** | ☐ | Conserver en l'état, pas adapté pour la migration à ce stade |
| **Retire** | ☐ | Décommissionner ; la fonctionnalité n'est plus nécessaire |
| **Rehost** | ☐ | Lift-and-shift vers une nouvelle infrastructure avec des changements minimaux |
| **Replatform** | ☐ | Lift-and-shift avec des optimisations ciblées (par ex., base de données gérée) |
| **Refactor** | ☐ | Re-architecturer les composants pour tirer parti des capacités de la nouvelle plateforme |
| **Replace** | ☐ | Remplacer entièrement par une nouvelle solution (par ex., produit SaaS) |

| Attribut | Détail |

| Méthode de migration des données | *[par ex., Export/Import, ETL, CDC, DMS, manuelle]* |
| Volume de données à migrer | *[par ex., 500 Go]* |
| Approche de basculement des utilisateurs finaux | One-off / Phased / Not applicable |
| Basculement des systèmes externes | One-off / Phased / Not applicable |
| Temps d'arrêt maximum acceptable | Zero / Seconds / Minutes / Hours / Days |
