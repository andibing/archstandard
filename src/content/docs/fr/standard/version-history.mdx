## Politique de versionnement

L'Architecture Description Standard (ADS) utilise le [**versionnement sémantique**](https://semver.org/) :

- **MAJOR** - Changements structurels qui cassent la compatibilité ascendante (par ex., sections réorganisées, supprimées ou fondamentalement modifiées)
- **MINOR** - Nouvelles sections, ajouts de guidance ou améliorations non-cassantes
- **PATCH** - Corrections, clarifications, corrections de coquilles

## Versions précédentes

Les versions majeures précédentes sont disponibles en permanence à des URLs versionnées. Les SADs référençant une version précédente restent valides.

## Versions publiées

### v1.0.0 - Version initiale

**Date :** 2026-03-27

La première version de l'Architecture Description Standard (ADS), synthétisant des années d'expérience pratique en architecture de solution avec des cadres établis :

- **ISO/IEC 42010** - Méta-modèle de description d'architecture (parties prenantes, préoccupations, points de vue, vues)
- **4+1 Architectural View Model** - Six vues architecturales (étendues à partir de 5 avec des vues Data et Security dédiées)
- **Cloud Well-Architected Frameworks** - Attributs de qualité dérivés des Well-Architected Frameworks d'AWS, Azure, GCP, Oracle et IBM
- **TOGAF** - Alignement des domaines d'architecture (Business, Data, Application, Technology)
- **Modèles SAD/HLD de l'industrie** - Bonnes pratiques issues d'arc42, ServiceNow, NHS SAF, ALMBoK et de modèles d'entreprise

**Structure :**
- 8 sections de premier niveau (0-7)
- 6 vues architecturales
- 5 attributs de qualité
- 3 niveaux de documentation (Minimum, Recommended, Comprehensive)
- JSON Schema pour la génération multi-format
- Traçabilité de l'alignement avec les cadres de référence

### v1.1 - Exemples et guidance

**Date :** 2026-03-31

- Quatre exemples de SAD complets (Employee Directory, Customer API Platform, Cloud Migration, archstandard.org)
- Guidance de notation (1/3/5) ajoutée à toutes les pages de section
- Améliorations d'accessibilité et de neurodiversité
- Traductions en français et en allemand
- Script de génération de modèles (schéma comme source unique de vérité)
