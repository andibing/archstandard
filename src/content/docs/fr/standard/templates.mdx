## Modèles téléchargeables

Téléchargez un modèle SAD vierge conforme à l'ADS v1.3.0 dans votre format préféré. Tous les modèles contiennent la structure complète des sections avec des champs à remplir.

### Formats lisibles par machine

#### Modèle Markdown

Modèle complet au format Markdown. Idéal pour le contrôle de version aux côtés du code dans les dépôts git.

**[Télécharger sad-template.md](/v1/templates/sad-template.md)**

#### Modèle YAML

Modèle structuré au format YAML. Idéal pour le traitement programmatique, l'intégration CI/CD et l'outillage.

**[Télécharger sad-template.yaml](/v1/templates/sad-template.yaml)**

#### Modèle JSON

Modèle structuré au format JSON. Se valide par rapport au JSON Schema ADS.

**[Télécharger sad-template.json](/v1/templates/sad-template.json)**

### Téléchargements du standard complet

#### Standard complet (DOCX)

Le standard ADS complet sous forme d'un document Word unique avec table des matières. Adapté à la lecture hors ligne, aux soumissions de gouvernance et à l'impression.

**[Télécharger ads-standard-v1.docx](/ads-standard-v1.docx)**

### Formats lisibles par l'humain

#### Word / DOCX

Générez un document Word à partir du modèle Markdown en utilisant [Pandoc](https://pandoc.org/). Idéal pour les soumissions formelles de gouvernance et la révision hors ligne.

Voir la commande Pandoc ci-dessous pour générer le document.

#### Confluence / Wiki

Copiez le modèle Markdown dans votre presse-papiers, puis collez-le directement dans Confluence, Notion ou tout wiki prenant en charge le Markdown.

Pour copier le modèle Markdown, téléchargez le fichier sad-template.md ci-dessus et copiez son contenu.

> **Astuce : Génération Word / DOCX**
>
> Convertissez le modèle Markdown au format Word en utilisant [Pandoc](https://pandoc.org/) :
>
> ```bash
> # Basic conversion
> pandoc sad-template.md -o sad-template.docx
>
> # With a custom reference document for branding/styling
> pandoc sad-template.md -o sad-template.docx --reference-doc=your-template.docx
> ```
>
> Un générateur de modèles Word dédié est prévu pour une future version du standard.

## Validation

Les modèles JSON et YAML peuvent être validés par rapport au [JSON Schema ADS](/v1/fr/standard/schema/) :

```bash
# Validate JSON template
npx ajv validate -s schema/ads.schema.json -d my-sad.json

# Validate YAML (convert to JSON first)
npx js-yaml my-sad.yaml | npx ajv validate -s schema/ads.schema.json -d /dev/stdin
```

## Personnalisation par l'organisation

Pour créer un modèle spécifique à votre organisation :

1. Téléchargez le modèle YAML ou JSON
2. Ajoutez votre section `organisationProfile` avec les mappings d'outils internes et les portes de gouvernance
3. Ajoutez les `customSections` requises par votre organisation
4. Distribuez le modèle personnalisé à vos équipes d'architecture

Voir [Conformité et utilisation](/v1/fr/standard/how-to-use/) pour les directives de personnalisation.
