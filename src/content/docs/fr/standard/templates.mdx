## Obtenir un modèle

Téléchargez un modèle SAD vierge depuis la page **[Téléchargements](/fr/standard/downloads/)** dans votre format préféré (Word, Markdown, YAML ou JSON).

## Trois façons d'utiliser les modèles

**1. Remplir manuellement** — ouvrez le modèle dans votre éditeur, votre wiki ou votre traitement de texte et complétez chaque section.

**2. Importer dans un LLM** — fournissez le modèle à ChatGPT, Claude ou Copilot avec un brief de projet pour générer un premier brouillon structuré.

**3. Coller dans Confluence** — copiez le modèle Markdown dans votre presse-papiers et collez-le directement dans Confluence, Notion ou tout wiki qui prend en charge le Markdown.

## Convertir entre formats

Utilisez [Pandoc](https://pandoc.org/) pour convertir entre formats :

```bash
# Markdown vers Word
pandoc sad-template.md -o my-sad.docx

# Markdown vers ODF (LibreOffice)
pandoc sad-template.md -o my-sad.odt

# Avec une charte personnalisée
pandoc sad-template.md -o my-sad.docx --reference-doc=your-template.docx
```

## Valider votre SAD

Validez un SAD complété (JSON ou YAML) par rapport au schéma ADS :

```bash
npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d my-sad.json
```

Voir la page [JSON Schema](/fr/standard/schema/) pour davantage d'options de validation.

## Personnaliser pour votre organisation

Vous pouvez étendre le standard pour votre organisation sans modifier les sections principales. Il existe deux points d'extension :

- **`organisationProfile`** — fait correspondre le standard à vos outils, standards et portes de gouvernance internes
- **`customSections`** — ajoute des sections spécifiques à l'organisation (par ex., listes de contrôle de gouvernance cloud, évaluations de confidentialité des données)

### Comment les ajouter à votre SAD

**En YAML ou JSON :** ajoutez les blocs `organisationProfile` et/ou `customSections` à la fin de votre fichier SAD (après `appendices`). Ils se placent au niveau supérieur, à côté des sections du standard.

**En Markdown :** ajoutez de nouveaux titres après la Section 7 (Annexes) avec votre contenu personnalisé. Utilisez le même format de tableau que le reste du document.

**En Word :** ajoutez de nouvelles sections après les Annexes. Suivez le même style de titre et de tableau.

**Pour toute votre organisation :** créez un modèle personnalisé avec le `organisationProfile` pré-rempli et distribuez-le à vos équipes d'architecture. Ainsi, chaque nouveau SAD démarre avec les bons mappings d'outils et portes de gouvernance.

### Exemple de profil d'organisation (YAML)

Ajoutez ce bloc à la fin de votre SAD YAML pour faire correspondre le standard aux outils et à la gouvernance de votre organisation :

```yaml
organisationProfile:
  organisationName: "Acme Corp"
  tooling:
    cmdb: "ServiceNow"
    secretStore: "HashiCorp Vault"
    monitoring: "Datadog"
    siem: "Splunk"
    cicd: "GitHub Actions"
    portfolioManagement: "LeanIX"
  internalStandards:
    - id: "STD-001"
      name: "Cloud Platform Standards"
      version: "2.0"
      mappedSections: ["physicalView", "securityView"]
    - id: "POL-008"
      name: "Information Security Policy"
      version: "4.1"
      mappedSections: ["securityView", "dataView"]
  governanceGates:
    - gateName: "Dev/Test Review"
      requiredMaturityLevel: "minimum"
      requiredSections: ["documentControl", "executiveSummary", "architecturalViews", "riskGovernance"]
    - gateName: "Production Approval"
      requiredMaturityLevel: "recommended"
      requiredSections: ["documentControl", "executiveSummary", "stakeholders", "architecturalViews", "qualityAttributes", "lifecycleManagement", "riskGovernance"]
    - gateName: "Enterprise Architecture Review"
      requiredMaturityLevel: "comprehensive"
      requiredSections: ["documentControl", "executiveSummary", "stakeholders", "architecturalViews", "qualityAttributes", "lifecycleManagement", "riskGovernance", "appendices"]
```

### Exemple de sections personnalisées

Ajoutez des sections spécifiques à l'organisation sans modifier le standard principal :

```yaml
customSections:
  - id: "cloud-governance"
    title: "Liste de contrôle de gouvernance cloud"
    description: "Exigences de gouvernance cloud spécifiques à l'organisation"
    maturityLevel: "recommended"
    parentSection: "physicalView"
    content: |
      | Contrôle | Statut | Preuve |
      |----------|--------|--------|
      | Politique de tagging appliquée | Oui / Non | |
      | Alertes de coût configurées | Oui / Non | |
      | Politique de sauvegarde appliquée | Oui / Non | |

  - id: "data-privacy"
    title: "Évaluation de la confidentialité des données"
    description: "Exigences spécifiques au RGPD et à la confidentialité des données"
    maturityLevel: "recommended"
    parentSection: "dataView"
    content: |
      | Question | Réponse |
      |----------|---------|
      | AIPD réalisée ? | Oui / Non |
      | Accords de sous-traitance des données en place ? | Oui / Non |
      | Processus de demande d'accès des personnes documenté ? | Oui / Non |
```

Voir [Conformité et utilisation](/fr/standard/how-to-use/) pour les directives de personnalisation complètes.
