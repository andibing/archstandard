## Modèle d'alignement

ADS utilise **ISO/IEC 42010** comme méta-modèle et se rapporte à plusieurs cadres établis. Cette page documente la traçabilité entre chaque section du standard et ses cadres sources.

## Alignement ISO/IEC 42010

Le standard suit le modèle conceptuel ISO 42010 :

| Concept ISO 42010 | Correspondance ADS |
|-------------------|---------------------|
| System of Interest | La solution documentée |
| Stakeholders | Section 2 : Parties prenantes et préoccupations |
| Concerns | Section 2 : Matrice des préoccupations |
| Architecture Viewpoint | Section 3 : Six points de vue définis |
| Architecture View | Le contenu de chaque Section 3.x |
| Model Kind | Diagrammes, tableaux et descriptions au sein des vues |
| Correspondences | Références croisées entre vues et références aux attributs de qualité |
| Architecture Description | Le document d'architecture de solution complet |

## Alignement avec le modèle de vues 4+1

| Vue originale 4+1 | Section ADS | Contenu clé |
|-------------------|---------------------|-------------|
| Logical View | 3.1 Vue logique | Architecture des composants, cartographie des services |
| Process View | 3.2 Vue intégration et flux de données *(adaptée)* | Flux de données, intégrations, interfaces |
| Development View | 5. Gestion du cycle de vie | CI/CD, SDLC, outillage |
| Physical View | 3.3 Vue physique (Infrastructure) | Déploiement, infrastructure, réseau |
| +1 Scenarios | 3.6 Scénarios | Cas d'utilisation, ADR |
| *(Étendue)* | 3.4 Vue données | Architecture des données (du point de vue Information RM-ODP) |
| *(Étendue)* | 3.5 Vue sécurité | Architecture de sécurité (transversale dans 4+1, élevée au rang de vue complète) |

:::note
Le standard étend le modèle classique 4+1 avec des vues dédiées **Données** et **Sécurité**. Dans le modèle 4+1 original, ce sont des préoccupations transversales réparties dans toutes les vues. L'architecture de solution moderne exige qu'elles soient traitées comme des vues de première classe en raison des exigences réglementaires, de conformité et de gouvernance des données. La vue Process a été adaptée pour se concentrer sur l'intégration et les flux de données plutôt que sur la concurrence.
:::
