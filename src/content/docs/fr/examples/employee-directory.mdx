## À propos de cet exemple

Il s'agit d'un Solution Architecture Document **fictif mais réaliste** pour un annuaire des employés interne chez Meridian Financial Services. Il illustre le standard ADS au niveau de documentation **Recommended**, approprié pour une application interne de Tier 4 qui passe en production.

Utilisez-le comme référence lors de la rédaction de votre propre SAD. La structure, les noms de champs et les valeurs d'énumération suivent exactement le standard.

| Caractéristique | Valeur |
|-----------------|--------|
| **Profondeur de documentation** | Recommended |
| **Criticité** | Tier 4 (Faible) |
| **Hébergement** | Azure PaaS |
| **Score global de conformité** | 3 / 5 |
