## Apprendre par l'exemple

Ces exemples de SAD montrent comment appliquer l'Architecture Description Standard à des solutions concrètes. Chaque exemple est un Solution Architecture Document complété (fictif) qui montre à quoi ressemble un bon document pour différents types de projets.

<Aside type="tip" title="Pas de vrais projets">
Ces exemples sont fictifs mais réalistes. Ils sont conçus pour illustrer les sections, les champs et la notation du standard — pas pour prescrire des choix technologiques spécifiques.
</Aside>

## Comment utiliser ces exemples

1. **Lisez celui qui correspond à votre type de projet** — choisissez l'exemple le plus proche de votre solution
2. **Notez la profondeur** — chaque exemple indique son niveau de documentation (Minimum, Recommended ou Comprehensive)
3. **Vérifiez la notation** — chaque exemple inclut un tableau de notation de conformité montrant comment il a été évalué
4. **Copiez la structure** — téléchargez un [modèle vierge](/fr/standard/templates/) et utilisez l'exemple comme référence en complétant votre propre SAD

## Valider votre SAD par rapport au schéma

Une fois votre SAD complété au format JSON ou YAML, validez-le par rapport au schéma ADS :

```bash
# Valider un SAD JSON par rapport au schéma hébergé
npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d my-sad.json

# Valider un SAD YAML (le convertir d'abord en JSON)
npx js-yaml my-sad.yaml | npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d /dev/stdin

# Avec Python
pip install jsonschema pyyaml
python -m jsonschema -i my-sad.json https://archstandard.org/schema/v1.0.0/ads.schema.json
```

Le schéma valide la structure, les champs obligatoires et les valeurs d'énumération — il garantit que votre SAD est conforme au standard. Voir la [page JSON Schema](/fr/standard/schema/) pour les détails complets.
