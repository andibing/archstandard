## Learning by Example

These example SADs demonstrate how to apply the Architecture Description Standard to real-world solutions. Each example is a completed (fictional) Solution Architecture Document showing what good looks like across different project types.

<Aside type="tip" title="Not Real Projects">
These examples are fictional but realistic. They are designed to demonstrate the standard's sections, fields, and scoring — not to prescribe specific technology choices.

## How to Use These Examples

1. **Read one that matches your project type** — pick the example closest to your solution
2. **Note the depth** — each example states its documentation depth (Minimum, Recommended, or Comprehensive)
3. **Check the scoring** — each example includes a compliance scoring table showing how it was assessed
4. **Copy the structure** — download a [blank template](/standard/templates/) and use the example as a reference while completing your own SAD

## Validating Your SAD Against the Schema

Once you've completed your SAD in JSON or YAML format, validate it against the ADS schema:

```bash
# Validate a JSON SAD against the hosted schema
npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d my-sad.json

# Validate a YAML SAD (convert to JSON first)
npx js-yaml my-sad.yaml | npx ajv validate -s https://archstandard.org/schema/v1.0.0/ads.schema.json -d /dev/stdin

# Using Python
pip install jsonschema pyyaml
python -m jsonschema -i my-sad.json https://archstandard.org/schema/v1.0.0/ads.schema.json
```

The schema validates structure, required fields, and enum values — ensuring your SAD conforms to the standard. See the [JSON Schema page](/standard/schema/) for full details.
